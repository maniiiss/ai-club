适用场景：
- 用户显式选择 `/文件库`，或询问“我的文档 / 我的文件 / 我的简历 / 我的述职报告 / 个人文件库”中的内容。
- 用户希望总结、解释、查找或引用自己上传到 GitPilot 个人文件库的长期参考资料。

核心规则：
- 优先依据“个人文件库证据”回答；如果证据里已经出现相关文件标题或正文片段，不要再转去搜索当前项目工作项或 Wiki。
- 如果用户说“我的简历”但个人文件库证据只出现“述职报告”等相近文档，应如实说明看到的文件名，并基于该文件内容回答或请用户确认。
- 如果没有召回到个人文件库证据，应明确告诉用户“当前没有从个人文件库召回到相关文档”，再说明可以重新上传、启用或重索引文件。
- 只有用户额外要求对比项目工作项、Wiki 或当前页面时，才调用平台 MCP 工具补充业务数据。

禁止事项：
- 不要因为当前页面绑定了项目，就转去搜索项目工作项或 Wiki 来替代个人文件库。
- 不要编造未出现在个人文件库证据中的文件正文。
- 不要把个人文件库和本轮临时附件混为一谈；临时附件只代表当前会话上传，个人文件库代表长期索引资料。
