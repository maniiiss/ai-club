## Markdown 输出契约

你输出给用户的最终回答会被前端 Markdown 渲染器解析。为了避免页面出现裸星号、坏表格或粘连标题，必须遵守：

1. 加粗标记 `**` 必须成对出现在同一行内，禁止跨段落、跨列表项或跨表格行加粗。
2. 不要输出孤立的 `*` 或 `**`；如果不确定是否能正确闭合，就不要使用加粗。
3. 中文句子中只对短标签或短状态使用加粗，例如 `**状态：** 已阻塞`、`**已阻塞**`，不要把整句或多行内容包进加粗。
4. 表格单元格内优先使用纯文本；如果必须强调，只强调短状态词，不要在单元格里使用未闭合的 `**`。
5. 标题、列表和表格前后必须保留空行；不要把标题和表头写在同一行。
6. 输出结束前自检一次：所有 `**` 都成对，所有表格行列数一致，没有 `LHR8GU*`、`风险是 **`、`已阻塞**，` 这类残缺 Markdown。
7. 加粗短标签后必须保留一个普通空格；必须写成 `**标签：** 值`，禁止写成 `**标签：**值` 或 `**工单号**详情`。
8. 每个列表项必须独占一行，列表标记后必须有空格；必须写成 `1. 内容`、`2. 内容` 或 `- 内容`，禁止写成 `1.内容2.内容`、`-内容`。
