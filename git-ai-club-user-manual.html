1. 系统概述

AI代理工程管理平台用于统一管理项目、迭代、工作项、Agent、模型、GitLab 自动合并以及 Jenkins 流水线， 适合需要把 AI 协作能力接入研发过程的团队。

项目协同

支持项目负责人、项目成员、迭代和工作项管理，适合按项目维度组织交付活动。

AI 执行链路

支持内置 Agent、提示词 Agent、HTTP API Agent 与 Agent Runtime 四种接入模式。

交付自动化

支持 GitLab 自动合并策略、AI Review 审核前置、Jenkins 构建触发与日志追踪。

平台既支持开发模式分别启动服务，也支持使用 Docker Compose 一次性启动完整环境。

3. 方式一：Docker Compose 一键启动

  1. 在项目根目录复制配置文件：Copy-Item .env.example .env
  2. 按需修改 .env 中的端口、GitLab 默认地址、Token 密钥等配置。
  3. 执行以下命令启动完整环境：
docker compose up -d --build

启动完成后，可通过浏览器访问前端地址 http://localhost:5173。

4. 方式二：开发模式分别启动

适合前后端联调和本地调试。

docker compose up -d postgres

cd backend
mvn -s maven-settings-central.xml spring-boot:run

cd ../frontend
npm install
npm run dev

cd ../code-processing
python -m venv .venv
.venv\Scripts\activate
pip install -e .
uvicorn app.main:app --reload --port 9000
注意： 如果需要重新初始化数据库，请执行 docker compose down -v 后再重新启动。 该操作会删除 PostgreSQL 数据卷中的现有数据。

首次使用

1. 默认管理员账号

账号 密码 来源
admin admin123 数据库初始化脚本创建，具备超级管理员角色

2. 登录流程

  1. 打开前端首页，进入登录页。
  2. 输入用户名和密码，点击“登录系统”。
  3. 登录成功后，系统默认进入“首页看板”。
  4. 登录状态保存在浏览器 localStorage 中，后端基于 Token 识别当前用户。

3. 首次登录建议操作

  • 在“用户管理”中修改或新增业务用户，并为其绑定角色。
  • 为需要查看个人 GitLab MR 告警的用户填写 GitLab 用户名。
  • 在“模型管理”中新增可用大模型配置，否则内置 Agent 和 AI Review 无法正常工作。
  • 根据实际环境补充 GitLab、Jenkins 等外部集成参数。

4. 权限行为说明

有权限：可正常进入页面与执行操作 无权限：前端提示后跳转到 403 页面

页面访问和按钮能力均受 RBAC 控制。例如只有具备 system:user:manage 的账号才能新增用户， 具备 cicd:manage 的账号才能维护 Jenkins 和触发构建。

数据库初始化后会内置以下演示内容，便于首次体验：

  • 项目：Agent Ops、Code Insight、Prompt Hub
  • Agent：Planner Agent、Coder Agent、Reviewer Agent
  • 任务：首页框架、任务流设计、代码扫描服务等示例数据

业务功能说明

1. 首页看板

首页会聚合平台项目数、我的任务、在线 Agent、自动合并告警，并给出任务中心、项目中心、自动合并、 CI/CD 和 Agent 管理等快捷入口。

  • “我的任务”优先显示与你直接相关的进行中或高优先级任务。
  • “待处理告警”重点显示自动合并失败、AI 拒绝或被跳过的 MR。
  • “在线 Agent”与“活跃项目”可作为日常巡检入口。

2. 项目管理

  • 支持按项目名、负责人、描述、状态筛选。
  • 创建项目时可选择负责人、项目成员、状态和项目说明。
  • 点击项目名称或“迭代管理”可直接进入该项目的迭代看板。
  • 删除项目时，关联 Agent 与任务会一并删除。

3. 迭代管理与工作项

迭代管理页面是项目执行的核心视图，左侧为迭代列表，右侧为工作项列表与迭代进度。

  • 支持新建迭代，配置名称、目标、状态、开始日期、结束日期、排序值和说明。
  • 支持“未规划工作项”视图，便于先建任务再排期。
  • 工作项类型支持“需求 / 任务 / 缺陷”。
  • 工作项支持设置负责人、协办人、所属迭代、执行 Agent、优先级、状态和 Markdown 说明。
  • 支持在列表中直接修改状态、优先级、负责人和协办人。
  • 删除迭代后，迭代下工作项会转为“未规划工作项”。
  • 页面内置项目燃尽图，可查看总工作项、已完成数和剩余数。

4. 任务管理

  • 支持按项目、执行 Agent、分类、优先级、状态进行筛选。
  • 任务支持绑定项目、执行 Agent、负责人和协作人。
  • 任务详细说明支持 Markdown，用于记录背景、目标和执行要求。
  • 若任务已绑定 Agent，可点击“运行 Agent”输入执行内容，并查看最近运行记录。
  • 运行记录会展示输入、输出、错误信息和执行人。

5. Agent 管理

平台支持四种 Agent 接入方式：

接入方式 适用场景 关键配置
内置 Agent 内置代码审查、测试建议、需求拆解 内置能力编码、模型配置、系统提示词
提示词 Agent 自定义 Prompt 逻辑 模型配置、系统提示词、用户提示词模板
HTTP API Agent 接入外部 HTTP 服务 接口地址、方法、认证、请求模板、响应路径
Agent Runtime 接入运行时网关，例如 OpenClaw Gateway 地址、Runtime 类型、Agent 标识、会话模板
  • Agent 可选择所属项目；为空时表示“全局 Agent”。
  • 支持测试 Agent，输入测试内容后查看输出结果。
  • 删除 Agent 后，关联任务会变成未分配状态。

6. 模型管理

  • 当前支持 OpenAI 与 Anthropic 两类提供商。
  • 新增模型时必须填写 API Key；编辑模型时可留空以保留原 Key。
  • 支持对模型做连通性测试，便于验证配置是否可用。
  • 模型配置是内置 Agent、提示词 Agent 和 GitLab AI Review 的基础依赖。

GitLab 与自动合并

1. GitLab 项目绑定

“项目绑定”用于把平台项目与 GitLab 仓库建立映射关系。

  • 配置项包括平台项目、GitLab API 地址、项目 ID 或路径、默认目标分支、API Token 和启用状态。
  • 新增绑定时必须输入 API Token；编辑时可留空保留原 Token。
  • 可执行“测试连接”以验证仓库是否可访问。
  • 可点击“查看 MR”预览当前仓库的合并请求。

2. 自动合并中心

自动合并支持两种执行模式：

  • 关联业务项目：复用 GitLab 绑定，适合和平台项目联动。
  • 独立运行：不依赖平台项目，直接配置 GitLab API、项目标识和 Token。

常用策略项

  • 源分支、目标分支、标题关键字过滤
  • 是否启用调度与 Cron 表达式
  • 是否要求 Pipeline 成功后合并
  • 是否 Squash、是否删除源分支、是否使用 GitLab Auto Merge
  • 是否开启 AI Review，以及选择哪个 Review Agent
  • 是否在合并后触发 Jenkins（仅关联业务项目模式可用）
关键限制： 独立运行模式不支持“合并后自动触发 Jenkins”。如果启用了 AI Review，则必须先配置可用的 Review Agent。

3. 自动合并日志

日志页可按结果和触发方式过滤执行记录，常见结果如下：

结果 含义
MERGED已成功合并
SKIPPED满足跳过条件，例如 Draft、冲突、规则不匹配等
AI_REJECTEDAI Review 判断风险较高，不建议自动合并
FAILED执行过程失败，例如接口连通性或权限问题
EMPTY没有找到符合条件的 MR

点击“详情”可查看合并日志详情、原因摘要以及 Markdown 格式的审查说明。

4. AI Review 工作方式

  • 后端会调用代码处理服务的审查能力，把 MR 标题、描述与变更 Diff 发送给大模型。
  • 审查结果会返回“是否通过、结论摘要、问题清单、Markdown 审查说明”。
  • 如存在高风险问题，系统会将该次自动合并标记为 AI_REJECTED。

CI/CD 管理与系统管理

1. Jenkins 服务管理

  • 配置 Jenkins 名称、地址、用户名、API Token、描述和启用状态。
  • 新增 Jenkins 服务时必须填写 API Token；编辑时可留空保留原 Token。
  • 支持“测试连接”检查服务连通性与 Job 数量。
  • 支持在抽屉中查看 Job 列表，并对单个 Job 执行手动构建。

2. 项目流水线绑定

  • 用于把平台项目映射到某个 Jenkins Job。
  • 配置项包括项目、Jenkins 服务、Job 名称、默认分支、构建参数 JSON、启用状态。
  • 保存时会校验 Job 是否存在。
  • 支持手动触发构建、查看最近构建状态、打开构建历史和控制台日志。

3. 代码处理服务

前端“代码处理”页面主要展示服务能力与处理流程，当前后端已接入调用链路。

接口 用途
GET /health健康检查
POST /api/code/summary扫描仓库结构，输出目录、文件类型和样本条目摘要
POST /api/code/review基于大模型执行代码审查，返回审查结论与 Markdown 说明

4. 用户管理

  • 支持维护用户名、昵称、邮箱、手机号、GitLab 用户名、启用状态和角色。
  • 新增用户时需要填写初始密码，长度至少 6 位。
  • 支持对已有用户执行“重置密码”。
  • 内置用户不可删除，部分字段也会受到限制。

5. 角色管理

  • 角色由“名称、编码、权限集合、启用状态、描述”构成。
  • 给角色配置权限后，用户只需绑定角色即可获得对应能力。
  • 删除角色会导致关联用户失去该角色对应权限。
  • 内置角色如“超级管理员”不可删除。

6. 功能权限管理

  • 功能分为 MENU 和 ACTION 两类。
  • 菜单权限可配置路由路径、组件名、图标和排序值。
  • 动作权限用于控制新增、编辑、删除、测试、触发构建等按钮能力。
  • 删除功能后，关联角色会失去该权限。
建议： 实际上线时，建议先设计“管理员、项目经理、开发、测试、运维”等标准角色，再为用户分配角色， 不建议直接把所有人都设为超级管理员。

常见问题

1. 登录失败

  • 确认前端、后端和数据库已经正常启动。
  • 默认管理员账号为 admin / admin123。
  • 如果数据库被重新初始化，原有用户和密码变更也会恢复到初始化状态。
  • 如果 Token 过期，重新登录即可，默认有效期为 12 小时。

2. 页面提示无权限或跳转到 403

  • 当前账号缺少页面或操作对应的权限码。
  • 请在“用户管理”确认用户已分配正确角色。
  • 请在“角色管理”确认角色已绑定所需功能权限。

3. GitLab 连接测试失败

  • 检查 GitLab API 地址是否正确，通常应以 /api/v4 结尾。
  • 检查项目 ID / 路径是否有效。
  • 检查 Token 是否具备读取 MR 和执行合并所需权限。
  • 确认后端服务器网络可访问 GitLab 地址。

4. 自动合并没有执行或没有结果

  • 确认策略已启用，且调度已开启或已手动点击“立即执行”。
  • 检查过滤条件是否过于严格，例如源分支、目标分支或标题关键字不匹配。
  • 草稿 MR、存在冲突的 MR、未满足 Pipeline 条件的 MR 会被跳过。
  • 如果启用了 AI Review，存在风险时会被拒绝。

5. 模型、Jenkins 或 Agent 测试失败

  • 检查 API 地址、用户名或 Token 是否填写正确。
  • 确认目标服务可从当前部署环境访问。
  • 编辑配置时如不打算替换密钥，可将密钥字段留空，系统会保留原值。

6. 首页看板没有显示我的 MR 告警

  • 请确认当前用户已填写个人 GitLab 用户名。
  • 请确认该用户有查看看板与 GitLab 模块的权限。
  • 若当前没有相关 MR 执行日志，看板会显示为空。

7. 附：建议上线前检查清单

  1. 修改 .env 中的默认数据库密码和 Token 密钥。
  2. 补充真实的 GitLab 默认地址、模型配置、Jenkins 配置。
  3. 创建正式用户和角色，避免长期使用默认管理员账号。
  4. 验证自动合并、Jenkins 构建和模型测试均已通过。
  5. 如需生产部署，建议接入 HTTPS、日志审计和数据库备份。
