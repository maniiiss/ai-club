# 公众端规划批量操作技术设计 v1

## 目标

公众端规划列表在一次用户操作中提交当前页已选择的最多 20 个工作项，避免浏览器对每个工作项分别发起 HTTP 请求。

## 接口契约

- `PUT /api/tasks/batch`：批量更新状态、优先级、负责人或所属迭代。
- `DELETE /api/tasks/batch`：批量删除工作项。
- `POST /api/public/tasks/batch-requirement-ai`：批量创建需求标准化或子任务拆解执行任务。

三个接口均返回逐项结果。单项权限、校验、余额或任务创建失败时，仅该项返回 `errorMessage`；已成功项保持生效。

## 执行与一致性

- 后端以单个 HTTP 请求接收整批 ID，去重后逐项调用既有领域更新、删除、积分和执行任务创建能力。
- 单条工作项更新保持原有完整载荷校验、通知、更新记录和知识图谱刷新语义。
- 批量 AI 每条需求独立扣费并保留独立执行任务；前端使用任务 ID 写入已有本地恢复记录，继续由单条 AI 助手审核结果。
- 请求上限与规划页当前页大小一致，为 20 项；不提供跨页全选。
