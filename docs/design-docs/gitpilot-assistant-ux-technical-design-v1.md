# GitPilot 公众端助手体验技术设计 v1

## 目标

本设计用于统一公众端 GitPilot 助手的回答展示、会话历史、记忆/文件库入口和聊天搜索行为。

## 回答元数据协议

助手模型在回答正文末尾追加一行隐藏标记：

```text
<!-- gitpilot-meta: {"title":"会话标题","followUps":["追问一","追问二"]} -->
```

后端通过 `AssistantResponseMetadata` 解析该标记：

- `content` 作为最终回答正文保存和返回；
- `followUps` 去重、裁剪到最多 3 条，写入会话展示态；
- `title` 仅在会话仍为默认标题且未手动重命名时生效；
- 标记缺失或 JSON 非法时，正文保留，追问为空，标题回退到首个问题裁剪值；
- 流式输出通过 `AssistantResponseStreamFilter` 延迟确认标记前缀，避免内部元数据闪现在页面。

服务端继续保留内部审计和排障数据，但不再把 `debug` 内容写入公众会话展示态或响应；旧请求中的 `debug` 参数仅为兼容而忽略。

## 会话历史搜索

新增 `GET /api/assistant/sessions/search`，仅允许搜索当前用户当前项目的会话，查询标题、最近预览和消息正文，默认同时覆盖活跃与已归档会话。

返回项包含会话标题、归档状态、命中角色、命中消息摘要和命中时间。前端以 250ms 防抖触发搜索，点击结果后切换归档视图并打开对应会话。

当前版本使用分页后的会话查询加单条命中消息查询，避免改变既有消息存储结构；后续数据量增长后再评估 PostgreSQL 全文索引或 trigram 索引。

## 前端入口布局

- `AssistantMemoryPanel` 只负责“记忆”和整理摘要；原“知识”入口统一改名为“记忆”。
- `AssistantFileLibraryPanel` 独立负责文件上传、搜索、启停、重新向量化、下载和删除。
- 顶部保留文件库图标；记忆和已归档入口收进 `...` 菜单。
- 左侧会话卡片只显示标题，不显示 `latestPreview` 副内容。

## 兼容与验证

- 保留手动重命名接口和 `titleCustomized` 语义，避免自动标题覆盖用户选择。
- 后端旧展示态 JSON 中的 `debug` 字段读取后会被忽略。
- 前端验证公众端测试和构建，后端验证 Assistant 服务测试、元数据解析测试、搜索作用域测试及编码检查。
