# 公众端发布观测交互稳定性设计

## 背景

公众端项目的“发布与观测”页面中，流水线中心搜索框聚焦时同时显示输入框自身的 focus ring 和全局 `:focus-visible` outline，视觉上形成双层高亮。流水线卡片的“详情、同步、编辑、删除”按钮共用条目级 `actionLoading` 状态，点击其中任意按钮时会连带替换“触发”按钮的图标和禁用状态，造成操作区短暂位移。

## 目标

- 搜索框聚焦时只保留一层清晰、稳定的主色焦点反馈。
- 详情、同步、编辑、删除操作只影响自身加载状态，不改变同一行其他按钮的尺寸或内容。
- 保持鼠标和键盘操作可用，并避免修改公共按钮组件和其他页面的既有视觉行为。

## 方案

在 `PipelinesPanel.tsx` 内将条目级加载状态改为动作级状态，使用包含条目类型、条目 ID 和动作名称的键标识当前请求。各异步处理函数设置对应动作键，操作按钮只根据自己的动作键决定 disabled 状态；“触发”继续使用公共 `Button` 的 loading 机制，但不会被其他操作连带触发。

搜索框显式关闭全局焦点 outline，继续使用输入框自身的主色 border 和 ring。流水线卡片中的原生文字按钮补充 `type="button"`，并使用 `focus-visible` 的非布局焦点反馈，避免浏览器默认按钮状态和全局 outline 叠加。

## 测试与验证

- 增加前端契约测试，检查流水线面板使用动作级 loading 标识、搜索框关闭全局 outline、原生操作按钮声明 `type="button"`。
- 先运行新增测试确认在修复前失败，再实现代码使其通过。
- 运行 `cd frontend-public && npm run test` 和 `cd frontend-public && npm run build`。
- 运行 `python scripts/check_encoding.py` 确认新增文档和源码保持 UTF-8 无 BOM。

## 非目标

- 不调整公共 `Button` 组件的全局 API 或其他页面按钮样式。
- 不改变流水线搜索请求、分页逻辑和任何后端接口行为。
