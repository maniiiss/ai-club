# 公众端技术设计 AI 弹窗加载稳定性设计

## 背景

公众端打开技术设计 AI 弹窗时，加载态内容高度小于加载完成后的表单高度，导致弹窗外框从小变大，产生明显布局抖动。

本次只优化公众端 `TechnicalDesignAiDialog`，暂不调整管理端工作项列表。

## 方案

- 弹窗桌面端打开时直接使用接近最终内容的稳定高度，同时限制最高不超过视口的 `90vh`。
- Header 与 Footer 保持固定，Body 始终占满剩余空间并独立滚动。
- Loading 作为 Body 内的覆盖层居中展示，不通过替换不同高度的根节点改变弹窗尺寸。
- 数据加载完成后，表单内容使用约 150–200ms 的透明度过渡显示，不执行高度动画。
- 小屏幕继续受 `90vh` 限制，避免固定高度超出可视区域。

## 异常与交互

- 加载失败仍显示现有错误信息，弹窗尺寸保持不变。
- Loading 期间提交按钮继续禁用，关闭按钮保持可用。
- 不延长请求时间，不引入人为等待或新的动画依赖。

## 验证

- 增加公众端源码契约测试，锁定弹窗稳定高度、覆盖式 Loading 和内容淡入结构。
- 运行 `frontend-public` 全量测试与生产构建。
- 运行编码检查和 `git diff --check`。
