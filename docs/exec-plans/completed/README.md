# 已完成的历史计划

本目录存放已完成的历史计划和归档方案，用于记录项目演进过程和经验沉淀。

## 文档来源

文档从 `../active/` 目录迁移而来，完成状态包括：

- 所有任务已完成并通过验收
- 方案目标已达成
- 项目已上线并稳定运行

## 归档要求

### 归档前检查清单

- [ ] 所有任务标记为已完成
- [ ] 实际完成日期已填写
- [ ] 关键成果已总结
- [ ] 经验教训已记录
- [ ] 相关技术债已关闭或更新状态

### 文档补充

归档时建议补充以下内容：

```markdown
## 完成总结

### 实际完成日期

yyyy-mm-dd

### 成果总结

- 成果1：描述
- 成果2：描述

### 经验教训

- 做得好的地方：描述
- 需要改进的地方：描述
- 可复用的经验：描述

### 关联资源

- 相关 PR：链接
- 设计文档：链接
- 测试报告：链接
```

## 目录结构

建议按时间或版本组织归档文档：

```
completed/
├── 2024-Q1/
│   ├── iteration-xxx.md
│   └── task-xxx.md
├── 2024-Q2/
│   └── ...
└── README.md
```

## 使用场景

1. **复盘会议** - 回顾历史方案执行情况
2. **新人入职** - 了解项目演进历程
3. **方案参考** - 借鉴历史方案经验
4. **问题追溯** - 追踪功能变更历史

## 注意事项

1. 归档文档保持只读，不再修改
2. 如需基于历史方案制定新方案，复制到 `active/` 目录
3. 定期清理重复或低价值的归档文档
4. 保留关键决策记录，即使方案已过时
