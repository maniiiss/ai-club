# 进行中的执行方案

本目录存放当前正在进行的执行方案、任务计划和迭代方案。

## 文档命名规范

推荐使用以下格式命名文件：

```
[类型]-[模块/功能]-[版本/日期].md
```

### 示例

- `iteration-v2024-Q2-project-management.md` - 项目管理模块迭代计划
- `task-gitlab-integration-phase2.md` - GitLab 集成第二阶段任务
- `plan-mobile-optimization.md` - 移动端优化执行方案

### 类型前缀

| 前缀 | 说明 |
|------|------|
| `iteration-` | 迭代计划 |
| `task-` | 具体任务计划 |
| `plan-` | 执行方案 |
| `sprint-` | 冲刺计划 |

## 文档模板

```markdown
# [方案名称]

## 基本信息

| 属性 | 内容 |
|------|------|
| 状态 | 进行中 |
| 负责人 | xxx |
| 开始日期 | yyyy-mm-dd |
| 预计完成 | yyyy-mm-dd |

## 目标

[方案目标描述]

## 任务清单

- [ ] 任务1
- [ ] 任务2
- [ ] 任务3

## 进度更新

### yyyy-mm-dd

- 完成情况描述
- 遇到的问题
- 下一步计划

## 风险与依赖

- 风险1：描述及应对措施
- 依赖1：描述及状态
```

## 注意事项

1. 方案完成后，将文件移动到 `../completed/` 目录
2. 文件名使用小写字母和连字符
3. 保持文档更新，记录关键进度
4. 关联相关技术债编号（如 TD-001）
