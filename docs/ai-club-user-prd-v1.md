# AI Club 用户侧 PRD v1

> 面向对象：产品负责人、项目负责人、研发负责人、测试负责人、平台管理员、实施与售前团队  
> 文档目标：用用户能理解的语言说明 AI Club 当前项目的核心功能、适用场景、业务价值和验收口径。  
> 范围说明：本文聚焦当前项目已具备或已有入口承载的核心能力，不展开底层技术实现细节。

## 1. 产品概述

AI Club 是面向软件研发团队的 AI 代理工程管理平台。它把项目管理、迭代协作、需求与任务、测试计划、代码仓库、流水线发布、运行观测、知识沉淀和智能体协作统一在一套平台中，让团队可以围绕一个项目持续完成“规划、开发、测试、发布、观测、复盘”的闭环。

平台的核心目标不是替代单一研发工具，而是把多套工具中的关键上下文收口到项目视角下，并通过 Hermes 对话式智能助手、执行中心、代码处理服务和模型管理能力，把 AI 代理真正接入日常工程流程。

## 2. 产品目标

1. 帮助研发团队在一个工作台中查看项目、迭代、工作项、测试、仓库、流水线和运行状态。
2. 支持团队把需求、测试、代码、发布和线上状态串成可追踪链路，减少跨工具切换。
3. 提供可控的 AI 辅助能力，包括需求分析、PRD 建议、API 测试用例生成、代码审查、自动化测试脚本生成和项目问答。
4. 建立平台级治理能力，包括权限、审计、通知、环境变量、模型配置、工具配置和快捷入口管理。
5. 支持对外只读分享，让项目相关方无需登录即可查看经过脱敏的项目发布与自动合并信息。

## 3. 用户角色

| 角色 | 主要诉求 |
| --- | --- |
| 平台管理员 | 配置用户、角色、权限、模型、工具、环境变量、服务器和全局规则，保障平台稳定运行。 |
| 项目负责人 | 管理项目成员、迭代、需求、任务、缺陷、仓库绑定、流水线和运行质量。 |
| 研发人员 | 查看分配任务、理解需求、发起或跟进 AI 执行任务、查看代码结构、触发构建和处理 MR。 |
| 测试人员 | 维护测试计划和测试用例，生成接口测试建议，执行自动化测试并查看报告。 |
| 运维/DevOps | 管理服务器、流水线、构建日志、运行实例、应用日志、健康检查和告警。 |
| 业务/外部相关方 | 通过只读分享页查看项目自动合并日志和流水线发布记录。 |

## 4. 核心业务流程

### 4.1 项目交付闭环

1. 管理员创建用户、角色和必要权限。
2. 项目负责人创建项目，维护成员、项目状态和基础信息。
3. 团队创建迭代，维护需求、任务、缺陷和评论。
4. 研发人员绑定 GitLab 仓库，查看代码结构、发起 MR、扫描仓库或配置自动合并。
5. 测试人员维护测试计划，生成测试用例，必要时将测试计划推送到 Gitee。
6. 团队通过执行中心发起 AI 任务、仓库扫描、开发执行或自动化测试。
7. DevOps 配置流水线，触发构建、查看运行历史和日志。
8. 项目上线后，在可观测性中心查看日志、健康趋势和运行实例。
9. 项目资料、复盘和知识沉淀进入 Wiki 与记忆事实图，供后续检索和 Hermes 问答。

### 4.2 AI 协作闭环

1. 用户在页面侧边打开 Hermes 对话抽屉。
2. Hermes 根据当前页面、用户、项目、Wiki 和历史记忆理解上下文。
3. 查询类问题优先通过平台工具读取真实数据。
4. 写操作默认生成待确认动作卡片，由用户确认后执行。
5. 对话结果可沉淀为用户会话记忆，后续继续召回。

## 5. 功能范围

### 5.1 首页看板

**用户价值**：为用户提供进入平台后的统一工作入口，快速看到项目概览、近期任务、常用系统入口和快捷操作。

**核心功能**：

- 展示项目、任务、智能体、快捷入口等概览卡片。
- 支持个人快捷任务配置。
- 支持常用系统访问入口配置和展示。
- 根据用户权限展示可访问的数据。

**验收口径**：

- 用户登录后能进入首页看板。
- 用户只能看到自己有权限访问的数据和入口。
- 快捷任务与快捷入口保存后再次登录仍可保留。

### 5.2 项目管理

**用户价值**：把团队研发活动收口到项目维度，作为后续迭代、任务、测试、仓库、流水线和观测能力的基础。

**核心功能**：

- 创建、编辑、删除和查询项目。
- 维护项目负责人、项目成员、状态、描述等基础信息。
- 作为项目级数据权限的核心载体，控制项目绑定资源的可见范围。
- 支持项目只读分享链接管理，用于外部查看脱敏后的项目公开信息。
- 支持进入项目迭代、记忆事实图、API 工作台、可观测性等关联能力。

**验收口径**：

- 有项目管理权限的用户可以维护项目。
- 项目参与者能查看自己可见项目及其绑定资源。
- 只读分享链接可生成、刷新、复制和失效。

### 5.3 迭代与工作项管理

**用户价值**：支撑项目团队按迭代组织需求、任务、缺陷和协作记录。

**核心功能**：

- 在项目下维护迭代。
- 创建和管理工作项，覆盖需求、任务、缺陷等协作对象。
- 支持负责人、状态、优先级、描述、评论、图片附件等协作信息。
- 支持 PRD 分析和建议应用，辅助用户完善需求内容。
- 支持需求侧开发和测试联动动作。
- 支持查看工作项关联的智能体运行记录。

**验收口径**：

- 用户可以在项目下进入迭代管理。
- 有权限的用户可以创建、编辑、删除工作项。
- 评论、附件和 AI 建议操作可在工作项详情中被查看和追踪。

### 5.4 Wiki 中心与知识沉淀

**用户价值**：沉淀项目知识、协作文档和可检索资料，减少项目知识散落在聊天、文档和仓库中的问题。

**核心功能**：

- 创建 Wiki 空间、目录和页面。
- 支持页面编辑、版本查看、版本恢复和同步。
- 支持文档导入预览和导入。
- 支持全文搜索、语义搜索和相关页面推荐。
- 支持空间成员管理。
- 支持空间或项目维度的记忆事实图查看。

**验收口径**：

- 用户能在 Wiki 中维护结构化页面。
- 页面修改后可查看历史版本并恢复。
- 搜索结果能返回用户有权限查看的页面。

### 5.5 API 工作台

**用户价值**：为项目维护接口资产，并在接口资产基础上提供 AI 测试用例建议，连接研发与测试协作。

**核心功能**：

- 以 API 项目列表作为入口，进入指定项目的 API 工作台。
- 通过 Yaade 嵌入式工作台维护接口集合、目录、请求和环境。
- 支持平台身份与 Yaade 会话打通，用户无需单独登录 Yaade。
- 支持从 GitLab 仓库同步 Spring API 到项目 API 集合。
- 支持选择单个接口生成 AI 测试用例建议。
- 保留历史 OpenAPI 导入导出、接口调试和调试记录能力的兼容接口。

**验收口径**：

- 用户进入 API 项目后能打开对应项目的 API 工作台。
- 用户只能访问自己有权限的项目 API 集合。
- AI 生成测试用例时不泄露敏感凭据，结果以人工审核建议形式呈现。

### 5.6 测试管理

**用户价值**：把测试计划、测试用例、自动化测试和外部测试平台同步统一管理。

**核心功能**：

- 创建、编辑、删除和查询测试计划。
- 在测试计划下维护测试用例和用例步骤。
- 支持测试计划关联项目和迭代。
- 支持测试计划与 Gitee 测试计划/测试用例手动推送。
- 支持将部分用例标记为 Playwright 自动化用例。
- 支持生成并验证自动化脚本，或执行已接入自动化。
- 自动化执行结果在执行中心中展示，包括报告、截图、Trace 等产物。

**验收口径**：

- 测试人员能维护测试计划和测试用例。
- 推送到 Gitee 后能记录远端绑定与最近推送结果。
- 自动化任务能在执行中心看到执行状态和产物。

### 5.7 智能体管理

**用户价值**：统一维护平台可用的智能体，为执行中心、Hermes 和业务自动化提供可配置执行能力。

**核心功能**：

- 创建、编辑、删除和查询智能体。
- 配置智能体名称、描述、能力和调用相关信息。
- 支持智能体测试，验证配置是否可用。
- 支持在任务或执行链路中关联智能体。

**验收口径**：

- 管理员能维护智能体配置。
- 配置完成后可发起测试并看到结果。
- 普通用户只能查看自己权限范围内可用的智能体。

### 5.8 执行中心

**用户价值**：把 AI 执行、仓库扫描、开发执行、自动化测试等异步任务统一调度、查看和追踪。

**核心功能**：

- 创建执行任务，查看执行任务列表、统计和详情。
- 支持取消、重试、确认执行计划等操作。
- 展示执行运行记录、步骤、事件、尾日志和执行产物。
- 支持通用任务、多仓开发执行、仓库扫描、测试自动化等执行类型。
- 执行完成后向发起人发送站内通知。

**验收口径**：

- 用户可查看自己有权限的执行任务和详情。
- 运行中任务能展示阶段状态和关键日志。
- 执行产物可下载或预览，失败任务有错误信息。

### 5.9 代码仓库管理

**用户价值**：将 GitLab 仓库与项目协作流程打通，让团队在平台内完成仓库绑定、代码理解、MR 辅助和自动合并治理。

**核心功能**：

- 维护项目 GitLab 仓库绑定。
- 支持当前用户 GitLab OAuth 授权。
- 支持读取分支、Tag、MR、用户候选等 GitLab 数据。
- 支持产品主线与产品分线管理，并发起主线到分线的同步 MR。
- 支持自动合并配置、日志查看和外部 Webhook 通知。
- 支持 AI MR 审查，历史问题带入、严格度门禁和审查缓存。
- 支持打开仓库代码结构快照和 GitNexus 全仓图。
- 支持仓库 API 同步、仓库扫描和规则集联动。

**验收口径**：

- 项目成员可查看有权限项目的仓库绑定。
- 有管理权限的用户可维护绑定、自动合并和产品分支配置。
- 自动合并日志记录每次处理结果、原因和通知状态。
- 代码结构入口能展示最近快照或触发全仓图打开。

### 5.10 Gitee 集成

**用户价值**：对接 Gitee 企业研发协作数据，让本地项目、迭代、工作项和测试计划能与 Gitee 保持人工可控同步。

**核心功能**：

- 在项目维度绑定 Gitee 项目。
- 在迭代维度绑定 Gitee 迭代。
- 按迭代手动同步 Gitee 工作项到本地任务。
- 保存 Gitee 工作项同步日志。
- 用户管理中维护本地用户与 Gitee 企业成员的映射。
- 测试计划详情页支持手动推送测试计划和测试用例到 Gitee。

**验收口径**：

- 绑定完成后可按迭代同步远端工作项。
- 同步结果可追踪，未匹配本地用户时不错误绑定人员。
- 测试计划推送只做新增和更新，不破坏远端人工数据。

### 5.11 流水线中心

**用户价值**：统一管理项目构建、发布和自动化触发，兼容内置 AI Club Pipeline 与外部 Jenkins。

**核心功能**：

- 在统一流水线中心查看 AI Club Pipeline 和 Jenkins 兼容绑定。
- 创建和维护 AI Club Pipeline，绑定项目、仓库、默认分支和固定触发变量。
- 内置 Woodpecker 作为默认执行底座，业务用户无需登录 Woodpecker。
- 当目标分支缺少配置文件时，通过内置模板生成配置 MR。
- 支持手动触发流水线。
- 支持维护 cron 定时触发。
- 支持公开 trigger webhook。
- 支持 callback webhook，在运行状态变化后向外部系统投递结果。
- 支持查看运行历史、聚合日志和最近运行摘要。
- 保留 Jenkins 服务实例管理、Job 列表、构建触发和日志查看。

**验收口径**：

- 用户能在同一个列表中看到项目相关流水线。
- 有构建权限的用户能触发构建。
- 有管理权限的用户能维护 cron、webhook 和配置补全。
- 运行历史与日志可在详情页按 provider 查看。

### 5.12 项目只读分享

**用户价值**：让业务方、客户或外部相关方无需登录平台，也能查看经过脱敏的项目发布相关信息。

**核心功能**：

- 项目列表提供只读分享入口。
- 支持生成、刷新、复制和失效分享链接。
- 公开页展示自动合并日志。
- 公开页展示项目流水线发布记录，聚合 AI Club Pipeline 和 Jenkins 摘要。
- 公开页字段严格脱敏，不暴露提交者邮箱、构建参数、控制台日志、Webhook Payload 等敏感信息。

**验收口径**：

- 有效链接可匿名打开。
- 失效或过期链接不可访问。
- 公开页只能查看对应项目的数据，不能越权查看其他项目流水线。

### 5.13 可观测性中心

**用户价值**：把项目上线后的运行日志、健康状态和实例配置纳入平台，帮助团队从项目视角排障。

**核心功能**：

- 展示项目可观测性列表。
- 在项目观测详情中查看运行实例、应用日志和健康趋势。
- 支持受管服务器实例和外部地址实例。
- 支持 SSH 日志采集和主动日志上报。
- 支持 HTTP/TCP 健康探针。
- 支持按项目、服务、级别、关键字和时间范围检索日志。
- 运行实例配置跟随项目权限控制。

**验收口径**：

- 用户只能查看有权限项目的日志和健康信息。
- 日志采集失败时能看到错误摘要，不影响其他实例展示。
- 健康趋势能反映最近探测结果。

### 5.14 服务器管理

**用户价值**：让平台管理员和运维人员在平台内管理部署服务器、资源指标、SSH 终端、SFTP 和告警。

**核心功能**：

- 创建、编辑、删除和查询服务器。
- 保存 SSH 登录方式、跳板机、标签、负责人和说明。
- 支持连接测试。
- 展示 CPU、内存、磁盘等资源指标历史。
- 支持连通性、CPU、内存、磁盘等告警配置和通知人。
- 支持浏览器内 SSH 终端。
- 支持 SFTP 文件列表、上传、下载、删除和新建目录。
- 支持运行期开关，关闭后隐藏入口并停用相关接口和调度。

**验收口径**：

- 服务器凭据不明文回显。
- 没有终端权限的用户不能打开 SSH 或 SFTP 功能。
- 资源指标和告警状态能在详情页查看。

### 5.15 模型管理与模型对比测试

**用户价值**：统一维护平台使用的模型配置，监控模型消耗，并通过基准测试比较不同模型表现。

**核心功能**：

- 创建、编辑、删除和查询模型配置。
- 支持不同模型类型和 OpenAI 兼容 provider 配置。
- 展示真实业务调用的今日 token 和本周 token。
- 不把测试模型调用计入业务消耗。
- 支持模型对比测试配置，选择 1 到 8 个聊天模型。
- 支持设置并发数、总请求数、流式开关、max_tokens、System Prompt 和 User Prompt。
- 展示失败率、平均输出、TTFT、平均耗时、P50、P95、Token/s 和吞吐等指标。
- 支持运行中取消和历史记录查看。

**验收口径**：

- 管理员能维护模型配置并测试可用性。
- 模型列表展示 token 统计。
- 对比测试结果可复现查看，运行中的测试不可被误删除。

### 5.16 Hermes 对话式协作

**用户价值**：为用户提供贯穿平台的 AI 协作入口，让 AI 基于当前页面和项目上下文回答问题、查询数据或生成待确认动作。

**核心功能**：

- 在平台内通过抽屉式入口发起对话。
- 支持文本问答和短语音转写后提问。
- 支持结合当前页面、项目、Wiki、用户和历史记忆回答问题。
- 查询事实时通过平台工具读取真实数据。
- 写操作默认返回动作卡片，由用户确认后执行。
- 对话成功后沉淀用户级会话记忆。

**验收口径**：

- 用户有 `hermes:chat` 权限时可使用 Hermes。
- Hermes 不应声称直接访问数据库。
- 涉及写入的操作必须先由用户确认。

### 5.17 记忆事实图与逻辑图谱

**用户价值**：把项目、Wiki 和 AI 协作中沉淀的事实关系可视化，帮助用户理解需求、知识和上下文之间的连接。

**核心功能**：

- 支持项目维度的记忆事实图。
- 支持 Wiki 空间维度的记忆事实图。
- 支持事实节点、关系和时间线等多种查看方式。
- 支持从事实图回到相关项目或 Wiki 上下文。

**验收口径**：

- 用户只能查看有权限的项目或 Wiki 空间事实图。
- 图谱为空时应给出清晰空状态。
- 节点详情能展示可读的事实摘要和来源信息。

### 5.18 自升级中心

**用户价值**：帮助平台团队发现系统自身的可优化点，形成巡检、建议、整改和执行的闭环。

**核心功能**：

- 维护自升级中心配置。
- 维护环境档案。
- 创建和管理巡检计划。
- 查看巡检运行记录。
- 管理优化建议，支持接受或拒绝。
- 生成和维护整改工作项。
- 支持手动发起巡检或整改执行。

**验收口径**：

- 自升级中心作为独立域，不跟项目成员权限绑定。
- 不同操作受自升级相关功能权限控制。
- 巡检和整改执行结果可追踪。

### 5.19 扫描规则集与仓库扫描

**用户价值**：让团队以统一规则扫描仓库质量问题，并把扫描结果纳入执行中心和治理流程。

**核心功能**：

- 管理仓库扫描规则集。
- 支持规则集校验。
- 用户或 Hermes 可选择 GitLab 仓库绑定和规则集发起扫描。
- 扫描任务进入执行中心。
- 代码处理服务执行扫描并回传结果。

**验收口径**：

- 规则集保存前可校验。
- 扫描任务可追踪状态、结果和产物。
- 用户只能扫描有权限项目绑定的仓库。

### 5.20 平台治理与系统管理

**用户价值**：为平台稳定运行、权限隔离、审计追踪和配置治理提供基础能力。

**核心功能**：

- 用户管理：维护账号、状态、密码、GitLab 用户映射、Gitee 成员映射。
- 个人中心：维护个人资料、修改密码、完成 GitLab OAuth 授权。
- 角色管理：维护角色、功能权限和数据权限。
- 功能管理：维护平台权限项。
- 工具配置：维护平台可调用工具。
- 环境变量管理：维护固定注册表内的运行时覆盖配置。
- 快捷入口管理：维护首页常用系统入口。
- PR 评审统计：按时间和开发组查看 PR 评审相关统计。
- 通知中心：接收执行、告警、任务等站内通知。
- 反馈管理：用户提交平台反馈。
- 操作日志：查看关键用户操作审计记录。
- 通用文件：统一处理头像、图片、Wiki 导入文档、快捷入口图标等文件上传和访问。

**验收口径**：

- 系统管理入口受功能权限控制。
- 敏感配置脱敏展示，不明文泄露。
- 操作日志能记录关键管理动作。

## 6. 权限与数据隔离

平台权限分为两层：

1. 功能权限：控制用户是否能进入页面、调用接口或执行动作。
2. 数据权限：控制用户在具备功能权限后还能看到哪些项目及项目绑定资源。

项目绑定资源默认跟随项目数据权限，包括项目、迭代、工作项、测试计划、执行中心、API 工作台、GitLab 绑定、CI/CD 流水线、可观测性和项目图谱等。

平台级独立资源主要依赖功能权限，包括模型管理、用户/角色/权限、工具配置、扫描规则集、操作日志、自升级中心和部分系统配置能力。

## 7. 通知与审计

平台需要在以下场景向用户提供明确反馈：

- 执行任务完成、失败或取消。
- 自动化测试完成并产生报告。
- 服务器告警触发或恢复。
- 自动合并、流水线或 Webhook 处理失败。
- 用户无权限、链接失效、配置缺失或外部服务异常。

平台需要保留关键管理动作的操作日志，包括用户、角色、权限、工具配置、环境变量、服务器、流水线、GitLab 配置等高风险操作。

## 8. 非功能需求

| 类别 | 要求 |
| --- | --- |
| 安全 | 敏感 Token、密码、私钥、Webhook 地址等必须密文存储或脱敏展示。 |
| 权限 | 所有核心页面和动作必须经过功能权限校验，项目绑定资源必须校验项目数据权限。 |
| 可用性 | 外部服务异常时，页面应展示可理解的错误或警告，不应导致整个工作台不可用。 |
| 可追踪 | 异步任务、自动合并、流水线、测试自动化和巡检执行必须有状态和日志可追踪。 |
| 兼容性 | 已存在的分享链接、历史 API 路由和 Jenkins 兼容能力应尽量保持可用。 |
| 移动端 | 新增业务页面优先复用既有移动端控制台布局、分页和交互规则。 |
| 编码 | 文档、源码、脚本统一使用 UTF-8 无 BOM，中文直接写入文件。 |

## 9. 成功指标

| 指标 | 说明 |
| --- | --- |
| 项目闭环使用率 | 项目是否同时使用迭代、工作项、测试、仓库、流水线和观测能力。 |
| AI 执行采纳率 | AI 生成的建议、脚本、审查结果或动作卡片被用户采纳的比例。 |
| 跨工具跳转减少 | 用户完成常见研发流程时，需要离开 AI Club 的次数是否下降。 |
| 执行可追踪率 | 异步任务是否都能在执行中心或对应模块看到状态和结果。 |
| 发布透明度 | 项目只读分享页和流水线记录是否能满足外部相关方查看诉求。 |
| 配置治理完整度 | 高风险配置是否纳入权限、脱敏、审计和固定注册表管理。 |

## 10. 当前边界与非目标

- AI Club 不直接替代 GitLab、Gitee、Jenkins、Woodpecker 或 Yaade，而是作为统一入口和编排层。
- Hermes 不直接访问数据库，必须通过受控平台工具获取事实。
- API AI 测试用例生成当前是建议型能力，不自动写回 Yaade 或测试计划。
- 模型对比测试的 usage 只用于指标计算，不计入业务 token 统计。
- 项目只读分享页只展示脱敏摘要，不开放构建日志、提交详情和敏感参数。
- 服务器管理是高风险能力，必须受运行期开关和权限控制。

## 11. 后续可演进方向

1. 把项目只读分享页升级为更完整的发布记录视图，支持版本号、关联 MR 和上线说明。
2. 将 API AI 测试用例建议与测试计划建立人工审核后的导入链路。
3. 扩展可观测性中心，补充更多健康维度和项目级 SLA 汇总。
4. 强化执行中心的执行模板和结果复盘能力。
5. 为自升级中心和仓库扫描增加更清晰的治理报表。
6. 沉淀用户侧帮助文档和典型角色操作手册。
